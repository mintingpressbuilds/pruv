# Cloud API Overview

The pruv cloud API at `https://api.pruv.dev` provides persistent storage, verification, sharing, and team access for your XY chains. All cloud features are optional -- pruv works fully offline.

## Base URL

```
https://api.pruv.dev
```

All API paths are prefixed with `/v1/`.

## Authentication

All API requests require a Bearer token:

```bash
curl -H "Authorization: Bearer pv_live_your_api_key_here" \
  https://api.pruv.dev/v1/chains
```

API keys use the prefix `pv_live_` for production and `pv_test_` for development. Get your key from the [pruv dashboard](https://app.pruv.dev).

See [Authentication](/cloud/authentication) for details.

## Response Format

All responses are JSON:

```json
{
  "id": "a1b2c3d4e5f6",
  "name": "my-chain",
  "length": 42,
  "root_xy": "xy_7e2f4a8c1d3b...",
  "head_xy": "xy_9c4e6b2d8a1f...",
  "created_at": "2025-01-31T15:30:00Z"
}
```

## Error Responses

Errors follow a consistent format:

```json
{
  "detail": "Chain not found"
}
```

| Status Code | Meaning |
|:-----------:|---------|
| `400` | Bad request -- invalid parameters |
| `401` | Unauthorized -- missing or invalid API key |
| `404` | Not found -- resource does not exist |
| `429` | Rate limited -- too many requests |
| `500` | Server error |

## Rate Limiting

Responses include rate limit headers:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 58
X-RateLimit-Reset: 1706745660
```

See [Rate Limits](/cloud/rate-limits) for details by plan.

## SDK Usage

Use the Python SDK instead of raw HTTP:

```python
from pruv import CloudClient

client = CloudClient(api_key="pv_live_your_key")

# Upload a chain
await client.upload_chain(chain)

# Verify remotely
result = await client.verify_chain("chain-id")

# List chains
chains = await client.list_chains()
```
